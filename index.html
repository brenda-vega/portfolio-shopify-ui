CASO 01 Mobile UI · E-commerce · Product Cards
Tokopedia Mobile Adaptation
Case study principal · Mobile-first UI exercise

Objetivo

Adaptar una experiencia de marketplace a una lectura mobile-first, priorizando navegación, producto, precio, confianza y acciones de compra.

Qué trabajé

  • Jerarquía visual para categorías, promos y producto
  • Componentes reutilizables para cards, banners y navegación
  • Sistema de spacing para mantener ritmo entre secciones
  • Patrones de product cards con foco en escaneo rápido

Criterio UI

  • Mobile-first thinking
  • Navegación inferior y accesos rápidos
  • Separación clara entre contenido comercial y utilitario
  • Conversión y UX visual sin saturar la pantalla

En mobile commerce, la conversión depende de una jerarquía que permita entender oferta, producto y acción en segundos.

Resultado: una adaptación más escaneable, modular y preparada para crecer con nuevas categorías o campañas.

CASO 03 Responsive UI · Design System · Layouts
Fidex Capital
UI Designer · Responsive product layouts · 2023-2024

Problema

Necesidad de una interfaz consistente entre desktop, tablet y mobile, con componentes documentados para desarrollo.

Qué hice

  • Design system en Figma
  • Componentes, estados y tokens visuales
  • Layouts responsivos para landing y contenido
  • Documentación de uso para handoff

Enfoque

  • Consistencia visual entre breakpoints
  • Grillas y spacing reutilizables

  • Arquitectura clara para escalar páginas

Resultado: componentes reutilizables y una base visual estable para construir más layouts.

CASO 04 Dashboards · Information Architecture · UI Systems
Kairos / Phronesis
Dashboard UI · Information architecture · 2023-2025

Problema

Organizar información compleja en dashboards claros, con navegación, estados y módulos fáciles de interpretar.

Qué hice

  • Sistema visual para dashboards
  • Arquitectura de información por módulos
  • Componentes para tablas, cards y estados
  • Versiones dark/light para lectura sostenida

Enfoque

  • Lectura rápida de indicadores
  • Navegación lateral y jerarquía por tareas
  • UI systems para producto complejo
  • Escalabilidad visual entre vistas

Los dashboards funcionan mejor cuando la arquitectura reduce ruido y guía la lectura de datos por prioridad.

Resultado: interfaces más ordenadas, consistentes y preparadas para múltiples flujos de información.
